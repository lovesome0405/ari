MARU 작업 정리
작성일: 2026-06-09
기준: 현재 프로젝트 파일, README.md, Git 커밋 로그를 확인해 정리함


1. 프로젝트 한 줄 요약

마루(MARU)는 외국인 방문객을 위한 모바일 우선 서울 전통문화 AI 가이드 정적 웹앱이다.

사용자는 언어를 고르고, 여행 조건을 선택한 뒤, 추천 전통문화 코스와 장소 설명, 지도 길찾기, 여행자 지원, 날씨 기반 추천, 저장 기록을 하나의 모바일 앱처럼 사용할 수 있다.


2. 핵심 방향

이 프로젝트의 핵심 구조는 README 기준으로 다음 3단계로 설명된다.

DB = 검증된 문화 사실
Server = 조건 필터링과 추천 점수
AI = 쉬운 문화 해설

현재 버전은 백엔드가 없는 정적 프론트엔드이므로, DB와 Server 역할 일부를 app.js와 JSON 데이터가 대신한다.
향후 Spring Boot + MySQL로 확장할 계획이 README에 정리되어 있다.


3. 현재 기술 구조

프레임워크 없이 HTML, CSS, JavaScript만 사용하는 정적 사이트다.

사용하지 않는 것:
- React
- Vue
- npm 빌드 과정
- Bootstrap
- Tailwind
- CDN
- 외부 이미지 에셋

사용하는 것:
- HTML 페이지 여러 개
- style.css 단일 스타일시트
- app.js 단일 앱 로직 파일
- ari_culture_resources_appjs.json 정적 문화자원 데이터
- manifest.json PWA 준비 파일
- service-worker.js 앱 셸 캐시
- GitHub Pages / Netlify / Vercel 정적 배포 설정

5. 사용자 흐름

1. index.html에서 언어를 선택한다.
2. 시작하기를 눌러 planner.html로 이동한다.
3. 여행 조건을 단계별로 선택한다.
4. routes.html에서 추천 코스를 확인한다.
5. route-detail.html에서 코스 상세와 장소별 문화 해설을 본다.
6. map.html에서 지도 서비스 링크와 이동 순서를 확인한다.
7. 필요하면 현재 위치를 허용해 가까운 문화 장소를 거리순으로 추천받는다.
8. support.html에서 여행자 지원 정보를 확인한다.
9. passport.html에 코스를 저장하고 문화여행 기록처럼 확인한다.

9. 추천 코스 생성 구조

초기에는 app.js 안에 고정 추천 코스 3개가 있었다.
이후 문화자원 JSON 데이터를 연결하면서, 데이터에서 코스를 생성하는 구조로 확장되었다.

현재 핵심 흐름:
1. app.js가 ari_culture_resources_appjs.json을 fetch한다.
2. 데이터를 normalizeCultureResource로 정규화한다.
3. 서울 장소만 필터링한다.
4. id 기준으로 중복을 제거한다.
5. 데이터가 충분하면 CULTURE_RESOURCES를 JSON 데이터로 교체한다.
6. ROUTE_BLUEPRINTS를 기준으로 ROUTE_DATA를 새로 생성한다.
7. JSON 로드가 실패하거나 데이터가 너무 적으면 app.js 안의 fallback seed data를 사용한다.

현재 route blueprint는 6개다.

첫 방문자 추천 코스
- 왕실 유물, 궁궐, 생활문화, 한옥, 전통거리를 한 번에 이해하는 입문 코스

비 오는 날 추천 코스
- 박물관, 체험 공간, 공연장을 중심으로 한 실내 중심 코스

사진 명소 코스
- 궁궐, 한옥마을, 한옥거리, 전통거리 중심의 사진/산책 코스

역사 깊이 보기 코스
- 왕실 의례, 궁궐, 도시사, 근현대사를 연결하는 학습형 코스

가볍게 걷는 코스
- 걷기 부담이 낮은 장소를 묶은 종로권 산책 코스

서울 남부 전통유산 코스
- 강남/송파권 왕릉, 사찰, 고분을 묶은 확장 코스

장소와 코스에는 외국인 방문객 관점의 판단 정보를 붙였다.

예시 필드:
- englishAvailable
- foreignerPopular
- reservationRequired
- languageSupport
- verifiedStatus
- sourceName
- sourceUrl
- qualityScore
- traditionScore
- foreignerScore
- hiddenScore
- rainyDayScore
- photoSpotScore
- walkingLevel
- indoorOutdoor

화면에서는 이 정보를 배지나 설명으로 보여준다.
예를 들어 영어 안내 가능 여부, 예약 필요 여부, 공식 확인 필요 여부, 외국인 인기 여부 등을 보여줄 수 있다.

주의:
현재 데이터는 정적 프로토타입용 seed data다.
운영 전에는 운영시간, 요금, 예약 여부, 주소, 좌표를 공식 출처로 다시 검증해야 한다.


11. 지도/길찾기 작업

현재 지도는 실제 지도 API를 임베드한 것이 아니라, 선택한 코스의 이동 순서와 외부 지도 서비스 링크를 제공하는 방식이다.

지원 지도 서비스:
- Google Maps
- 네이버지도
- 카카오맵

좌표가 있는 장소:
- 좌표를 우선 사용해 링크를 만든다.

좌표가 없는 장소:
- 장소명 + 서울 검색어를 사용한다.

map.html에서는 선택한 지도 서비스를 저장하고, 이후에도 같은 서비스를 기본값으로 사용할 수 있다.


12. GPS 근처 추천 작업

최신 커밋에서 GPS 기반 가까운 장소 추천 기능이 추가되었다.

구현된 내용:
- navigator.geolocation.getCurrentPosition 사용
- enableHighAccuracy: true
- timeout: 8000ms
- maximumAge: 60000ms
- 사용자의 현재 위치는 가까운 장소 계산에만 사용
- localStorage에 저장하지 않음
- Haversine 거리 계산으로 장소까지의 거리 km 계산
- 좌표가 있는 문화자원만 대상으로 거리순 정렬
- 가까운 장소 추천 결과에 거리 배지 표시
- 위치 정확도도 안내 문구로 표시

예외 처리:
- 위치 권한 거부
- 위치 확인 실패
- 브라우저가 geolocation 미지원
- 좌표가 있는 장소 데이터 없음

예외 상황에서는 기본 서울 추천 코스를 보여주는 fallback 상태로 돌아간다.

주의할 점:
map.html과 다국어 번역 문구는 현재 GPS 근처 추천 기능에 맞게 정리되었다.
위치 정보는 가까운 장소 추천에만 사용되며 저장하지 않는다는 안내를 유지했다.


13. 홈 화면 시간대 이미지 작업

홈 화면은 서울 시간대 Asia/Seoul을 기준으로 시간대를 계산한다.

이미지 세트:
- 낮/day 이미지 4장
- 저녁/evening 이미지 4장
- 밤/night 이미지 4장

아침과 점심은 day 이미지 세트를 공유한다.
이미지는 일정 간격으로 회전하며, CSS 변수 --time-bg-image를 통해 배경에 적용된다.

이미지 파일이 없을 경우 깨진 이미지 아이콘 대신 MARU placeholder가 나오도록 처리되어 있다.


14. 이미지 정책과 프롬프트 작업

image-prompts.md에 AI 이미지 생성용 프롬프트가 정리되어 있다.

정책 방향:
- 외부 저작권 사진을 무단 사용하지 않음
- 실제 장소 사진으로 오해되지 않게 "AI 생성 이미지 / 이해 보조 이미지" 성격을 명시
- 실제 운영에서는 공식 사진 권한 또는 자체 촬영 이미지를 별도로 확보해야 함

프롬프트 종류:
- 추천 코스 이미지
- 장소 이미지
- UI theme hero 이미지


15. 아리 챗봇 작업

아리(ARI)는 마루 앱 안의 전통문화 안내 챗봇이다.

구현된 내용:
- 앱 전역에 floating chat UI로 표시
- ARI 캐릭터 이미지 assets/images/ari/ari1.webp 사용
- 빠른 질문 버튼 제공
- 사용자가 입력한 질문에 답변
- Ollama local AI가 있으면 연결 시도
- 연결 실패, CORS 문제, Ollama 미실행 상황에서는 fallback 답변 사용

fallback 답변은 다음 데이터를 바탕으로 만든다.
- ROUTE_DATA
- CULTURE_RESOURCES
- WEATHER_DATA
- SUPPORT_DATA

중요 원칙:
아리는 검증되지 않은 사실을 새로 만들어내는 것이 아니라, 앱 안의 문화자원 데이터와 코스 데이터를 쉽게 설명하는 역할이다.

- 서울의료관광센터
- 비상 연락처
- 서울관광불편신고센터

17. 저장/패스포트 기능

passport.html은 사용자가 저장한 코스와 문화여행 스탬프를 보여준다.

저장 방식:
- localStorage 사용

저장되는 것:
- 선택한 코스
- 저장한 코스 id 목록
- 언어 설정

21. 현재 상태에서 남은 확인사항

2. route/place 이미지 확충 필요
- 홈 이미지는 들어가 있지만, 추천 코스/장소별 실제 이미지 asset은 아직 부족해 placeholder에 의존할 수 있다.
- 장소별 실제 이미지는 향후 공식 이미지 권한 확보 또는 자체 제작 이미지로 보강하는 것이 좋다.

3. PWA PNG 아이콘 추가는 선택 사항
- 현재 manifest는 실제 존재하는 SVG 아이콘을 참조한다.
- 192px/512px PNG 아이콘은 앱 설치 품질을 높이기 위한 향후 보강 항목이다.

4. 실제 백엔드 연동은 아직 계획 단계
- Spring Boot + MySQL 테이블과 API 계획은 README에 있지만, 현재 저장소에는 백엔드 코드는 없다.

5. 실제 날씨 API는 아직 미연동
- weather.html은 구조와 정적 추천을 보여주는 단계다.

6. 지도 API 임베드는 아직 미구현

22. 이 프로젝트를 발표할 때 설명하기 좋은 구조

문제:
외국인 방문객은 서울 전통문화 장소를 단순 관광지 목록으로만 접하기 쉽고, 장소 사이의 문화적 연결과 방문 난이도, 언어 지원, 예약 필요 여부를 한 번에 파악하기 어렵다.

해결:
마루는 궁궐, 박물관, 한옥, 전통시장, 공연, 왕릉 등을 스토리형 코스로 묶고, 외국인 방문객에게 필요한 언어/편의/지도/지원 정보를 모바일 앱 흐름으로 제공한다.

핵심 차별점:
- 맛집/명소 나열이 아니라 전통문화 이야기 흐름으로 추천
- 문화자원 데이터와 추천 점수를 분리해 향후 백엔드 확장 가능
- AI는 사실을 생성하는 역할이 아니라 검증된 문화 정보를 쉽게 설명하는 역할
- 네이버지도, 카카오맵, Google Maps를 모두 고려
- 위치 기반 가까운 전통문화 장소 추천까지 확장
- 다국어 전환과 외국인 지원 정보를 앱 전체에 적용
